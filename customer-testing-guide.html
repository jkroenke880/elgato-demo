4. How to Test

Looking Up a Profile via the API

You can query any pre-seeded user's resolved profile using the Segment Profile API:

curl https://profiles.segment.com/v1/spaces/{space_id}/collections/users/profiles/email:{email}/traits \
  -u "<profile_api_token>:"

Replace {space_id} with the relevant space ID from Section 2, {email} with any of the test user emails from Section 3, and use the Profile API token as the Basic Auth username with a blank password (the trailing colon in -u "token:" is required).

Generating Your Own Events via the Demo Sites

  1. Open either demo URL in your browser.
  2. Browse products and add items to your cart.
  3. Proceed to checkout and enter any email address — this fires an identify call that ties your anonymous browsing session to a known profile.
  4. Complete the order — this fires Order Completed and populates purchase-based audiences.
  5. Check the Events widget (bottom-left) to see every call fire in real time.
  6. To test cross-brand stitching, use the same email address on both the Corsair and Elgato sites, then look up your profile in the Combined Profiles space.

Testing Cross-Brand Identity Resolution

  1. Go to the Corsair site, add something to cart, checkout with your email.
  2. Go to the Elgato site, add something to cart, checkout with the same email.
  3. Wait ~2–3 minutes for Unify to process.
  4. Query the Combined Profiles space using your email — you'll see a single stitched profile with events from both sources, and the cross_brand_buyer audience trait set to true.

5. Testing with Postman

Postman is the easiest way to send raw events and query profiles without touching the demo sites. The two sections below cover everything you need: sending data in via the Segment HTTP Tracking API, and pulling audience membership out via the Profile API.

5a. Sending Events — Segment HTTP Tracking API

The Tracking API accepts identify, track, and page calls. Authentication is Basic auth with the write key as the username and an empty password.

FieldValue
Base URLhttps://api.segment.io/v1
MethodPOST

Tip: Set up two environments in Postman — one for Corsair and one for Elgato — each with a writeKey variable. Use {{writeKey}} in the Authorization tab so you can switch sources without editing every request.

Example 1 — identify (create or update a user profile)

Send this to POST https://api.segment.io/v1/identify to create a new user or update traits on an existing one. This is what fires on checkout in the demo sites.

{
  "userId": "test.user@yourcompany.com",
  "traits": {
    "email":     "test.user@yourcompany.com",
    "firstName": "Test",
    "lastName":  "User",
    "city":      "San Francisco"
  },
  "timestamp": "2026-05-20T12:00:00Z"
}

Example 2 — track: Product Viewed

Send to POST https://api.segment.io/v1/track. Use the Corsair write key to populate the corsair_keyboard_shopper audience, or the Elgato write key for stream_deck_interested.

  "event": "Product Viewed",

The Profile API lets you look up a resolved user profile and see which audiences they belong to. Authentication uses Basic Auth with the Profile API access token as the username and an empty password.

Profile API tokens are pre-configured for this demo environment. Use the tokens from the Reference table in Section 6 — one per space, as Basic Auth username with a blank password.

Profiles can be looked up by any known identifier. The most useful for this demo is email:

Example 6 — Get all traits for a user (including audience membership)

Audience membership is stored as boolean traits on the profile. A value of true means the user is in that audience.

Recommended Postman workflow:
  1. Create a Collection called Segment Demo with two folders: Send Events and Profile API.
  2. Create two Environments: Corsair and Elgato, each with variables writeKey and profileToken and spaceId.
  3. Add a third environment Combined with the Combined Profiles spaceId and its own profileToken.
  4. Send Examples 1–3 with the Corsair environment, then Examples 1 & 4 with the Elgato environment using the same email.
  5. Wait ~2 minutes, then run Examples 7 & 8 in the Combined environment to see the stitched cross-brand profile.
